Password Cracking

What is a Password?

A password is any string of characters used to authenticate someone's identity.

Password Strength: The strength of a password depends on factors like the length and word choice (i.e. If a word in the dictionary is used for the password).

Current Systems: Most systems now are safe due to added measures of security including: limits on password attempts, two-factor identification, and if-failure security questions.

Poor passwords: Aspects of weak password choices include: small number of characters, use of real words, and using names or dates relating to a person's life.

Types of Attacks


- A brute force attack primarily consists of configuring predetermined values, making requests to a server using those values, and then checking the response.

- These attacks are usually sent via GET and POST requests.

- This works by taking a word list of known pages and simply requesting each known page then analyzing the HTTP response to determine if the page exists on the target server.



- A dictionary attack is a method of breaking into a password-protected computer or server by systematically entering every word in a dictionary.

- A simple form of dictionary attack is used by email spammers, who simply send a message to email addresses consisting of names (using a dictionary) followed by the '@' symbol and the name of a domain.

Prevention

Random Combinations: Password attack prevention has become very secure in recent years. Some systems employ the use of random combinations of letters, both upper and lower case, as well as numbers and symbols. With these combination methods it becomes a very hard and tedious process to crack passwords.

Attempt Limits: This method of attack prevention is one of the most successful, reducing the possibility of brute force or dictionary attacks being successful by a great deal. By limiting the attempts allowed to enter a password in a given amount of time, the ability of a brute force attack is completely stopped.

Captcha: This is a system which distinguishes humans from machines by requiring entering distorted letters and / or characters presented on the screen which are unreadable by a machine.


Character Requirement: Requiring the user to create a password with more than 8 characters greatly increacses the safety of an account. This is because the amount of time to check a dictionary increases exponentially as you add more characters to a password, rendering brute force and dictionary attacks virtually useless.
